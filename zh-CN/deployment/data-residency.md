# 数据驻留与地域

Routero AI 可在多个 AWS 地域运行。对于大多数客户，Routero Cloud（新加坡）已经足够。受监管市场——尤其是有数据本地化要求的市场——应选择单租户云或私有部署，并选定合适的地域。

---

## 可用地域

| 地域 | AWS 地域 | 提供方式 |
|---|---|---|
| 新加坡（默认） | `ap-southeast-1` | Routero Cloud、私有部署 |
| 美国东部 | `us-east-1` | 单租户云、私有部署 |
| 美国西部 | `us-west-2` | 单租户云、私有部署 |
| 欧洲西部 | `eu-west-1` | 单租户云、私有部署 |
| 欧洲中部 | `eu-central-1` | 单租户云、私有部署 |
| 亚太东南部 | `ap-southeast-1` / `ap-southeast-2` | 单租户云、私有部署 |
| 中国北京 | `cn-north-1`（光环新网 Sinnet） | 仅单租户云 |

{: .note }
**仅限企业版套餐：** 单租户云的地域选择在开通时与你的解决方案工程师共同确定。

## AWS 中国（北京）部署

Routero 在 AWS 中国北京（`cn-north-1`，光环新网 Sinnet 账户）运营一套专用的生产环境堆栈，面向受中国《个人信息保护法》（PIPL）约束或需要境内数据驻留的客户。

**与全球堆栈的主要区别：**

- **独立的 AWS 账户与状态** —— 中国堆栈与全球基础设施完全隔离。
- **境内 UI** —— 控制台由位于北京的 ECS 任务提供（而非 Cloudflare，后者在中国大陆的覆盖有限）。
- **ECR 镜像镜像同步** —— 容器镜像被同步到中国的 ECR 仓库；不存在出境的镜像拉取流量。
- **独立的 DNS 根域** —— `cn.routero.ai`（UI）与 `cn-platform.routero.ai`（API/平台）。
- **ICP 合规** —— 该部署需要 ICP 备案。一项已注册 ICP 的主机名迁移正在进行中；可用性取决于 Sinnet 端口开通与 ICP 审批。
- **PIPL 合规对齐** —— 在中国堆栈内处理的数据不会经由国际网络路径传输。

如需了解中国部署的可用性与接入流程，请联系你的解决方案工程师。

## “数据驻留”在 Routero 中的含义

| 数据类型 | Routero 存储什么 | 存储位置 |
|---|---|---|
| 提示词/响应内容 | **不存储**（在内存中处理后即丢弃） | — |
| 审计日志（元数据） | Token 数量、模型、成本、延迟、用户密钥、组织、时间戳 | 你所在地域的 RDS |
| 护栏违规 | 违规类型与消息（不含被拦截的内容） | 你所在地域的 RDS |
| 供应商 API 密钥 | 在 RDS 中静态加密存储 | 你所在地域的 RDS |
| 记忆会话内容 | 向量嵌入 + 检索到的事实（若启用记忆即服务） | 你所在地域的 Postgres + 可选的 Neo4j |
| 支出与用量 | 每请求成本，按密钥/团队/组织汇总 | 你所在地域的 RDS |

{: .note }
在私有部署中，以上所有内容都存放在你自己的数据库中。在 Routero Cloud 中，它们存放在 Routero 位于新加坡的 RDS 中。在单租户云中，它们存放在你所选地域的专用 RDS 中。
